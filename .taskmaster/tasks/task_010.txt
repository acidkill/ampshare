## Usage Guide

### Registering a New Web Component

```typescript
// Import the registration function
import { registerWebComponent } from './web-components/registry';

// Define your component
class MyComponent extends HTMLElement {
  // Component implementation
}

// Register it
registerWebComponent('my-component', MyComponent);
```

### Lazy Loading Components

```typescript
import { lazyLoadComponent } from './web-components/lazy-registry';

// Load a component when needed
lazyLoadComponent('my-lazy-component', () => import('./path/to/component'));
```

## Build and Deployment

### Local Development

```bash
# Install dependencies
npm install

# Start development server
npm run dev
```

### Production Build

```bash
# Build Docker image
docker build -t ampshare:latest .

# Run container
docker run -p 3000:3000 ampshare:latest
```

## Rollback Procedures

### Rolling Back a Deployment

1. Identify the previous stable version tag
2. Deploy the previous version:

```bash
# Pull the previous version
docker pull ampshare:previous-version

# Stop the current container
docker stop ampshare-current

# Start the previous version
docker run -d --name ampshare-rollback -p 3000:3000 ampshare:previous-version
```

3. Verify the rollback was successful by testing the application
4. Update monitoring and alerts

### Code Rollback

1. Create a revert commit in git:

```bash
git revert HEAD~1..HEAD
```

2. Push the revert commit
3. Let the CI/CD pipeline deploy the reverted code
```
